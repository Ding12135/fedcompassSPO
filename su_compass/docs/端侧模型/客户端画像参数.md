# 客户端画像参数

本文档整理 `su_compass` 端侧模型中五类标准客户端的画像参数，对应代码位置：

- 画像定义：`su_compass/runtime/profile.py` 中的 `build_default_profiles()`
- 虚拟运行全局参数：`su_compass/runtime/virtual_runtime.py` 中的 `VirtualRuntimeModel`

## 1. 五类客户端概览

| 类型 ID | 中文名 | 设计定位 | 主要区分维度 |
|---------|--------|---------|-------------|
| `stable_fast` | 稳定快 | 理想基线客户端 | 高算力、高带宽、低波动 |
| `stable_slow` | 稳定慢 | 计算受限但可靠 | 低 `compute_capacity`、低波动 |
| `compute_volatile` | 计算波动 | 共享算力/后台争用 | 高 `compute_jitter_cv`、偶发卡顿 |
| `network_poor` | 网络差 | 跨机构弱网/VPN | 低带宽、高网络波动、高延迟 |
| `availability_limited` | 可用性受限 | 移动/边缘设备 | 低 `availability_probability`、高卡顿 |

论文依据：

- **FedScale**：计算能力、网络带宽、运行波动
- **FLASH**：客户端可用性
- **FedCompass**：速度感知调度下的快慢客户端差异

## 2. 全局虚拟运行参数

以下参数不在 `ClientRuntimeProfile` 中，由 `VirtualRuntimeModel` 统一设置，五类客户端共用：

| 参数 | 默认值 | 单位 | 含义 |
|------|--------|------|------|
| `base_step_time` | 0.05 | 秒/步 | 计算能力为 1.0 时，每个 local step 的基础训练耗时 |
| `model_size_mb` | 5.0 | MB | 服务器下发模型大小 |
| `update_size_mb` | 5.0 | MB | 客户端上传更新大小 |
| `unavailable_delay_mean` | 5.0 | 秒 | 设备不可用时的平均等待时间（指数分布均值） |
| `seed` | 2026 | — | 随机种子，保证模拟可复现 |

单轮耗时分解：

```text
round_time =
    download_time
  + train_time
  + upload_time
  + spike_delay
  + availability_wait

train_time = local_steps × base_step_time / compute_capacity
download_time ≈ model_size_mb × 8 / download_bandwidth_mbps + base_latency
upload_time   ≈ update_size_mb × 8 / upload_bandwidth_mbps + base_latency
```

## 3. 画像参数字典

| 参数 | 类型 | 含义 |
|------|------|------|
| `compute_capacity` | float | 计算能力，越大训练越快 |
| `compute_jitter_cv` | float | 计算时间波动系数，越大越不稳定 |
| `upload_bandwidth_mbps` | float | 上传带宽（Mbps） |
| `download_bandwidth_mbps` | float | 下载带宽（Mbps） |
| `network_jitter_cv` | float | 网络耗时波动系数 |
| `base_latency` | float | 基础网络往返延迟（秒） |
| `spike_probability` | float | 每轮偶发卡顿触发概率，范围 [0, 1] |
| `spike_delay_mean` | float | 触发后卡顿时长的指数分布均值（秒） |
| `availability_probability` | float | 每轮设备可用概率，范围 [0, 1] |

随机采样说明：

- **是否卡顿 / 是否可用**：`random() ~ Uniform(0, 1)` 做伯努利判断
- **卡顿时长 / 不可用等待**：触发后从 `Exponential(mean)` 采样

## 6. 推荐八客户端映射

后续接入 FedCompass 主实验时，建议将五类画像映射到 8 个客户端实例：

| 客户端 ID | 画像类型 | 数量 |
|-----------|---------|------|
| `client_0`, `client_1` | `stable_fast` | 2 |
| `client_2` | `stable_slow` | 1 |
| `client_3`, `client_4` | `compute_volatile` | 2 |
| `client_5`, `client_6` | `network_poor` | 2 |
| `client_7` | `availability_limited` | 1 |

同类型客户端使用相同画像参数，实例间差异由不同随机种子产生。

## 7. 验证命令

默认五类画像验证：

```bash
python -m su_compass.runtime.simulator --mode default
```

单客户端可调验证：

```bash
python -m su_compass.runtime.simulator \
  --mode single \
  --client_id custom_slow \
  --compute_capacity 0.3 \
  --compute_jitter_cv 0.05 \
  --upload_bandwidth_mbps 30 \
  --download_bandwidth_mbps 60
```
