# CIFAR-10 数据划分与公平对比规范

本文档是后续 CIFAR-10 实验的数据划分与公平性冻结规范。若其他说明与本文冲突，以本文为准；变更本文中的冻结项前，必须记录原因、代码版本和新的实验编号，并让 FedCompass、Trust-Q 与 Q+Group 全部重新运行。

## 1. 研究目标与基本原则

本项目研究动态端侧状态造成的完成时间预测误差，以及 StateCompass 的状态感知 Q 分配与状态时间窗分组。主要比较链为：

```text
FedCompass
→ StateCompass Trust-Q V2
→ StateCompass Q + Group
```

论文需要证明的核心不是通过更换数据划分让新方法获得更高的最终精度，而是：

> 在相同数据、模型、优化器、聚合器、训练预算和端侧画像下，StateCompass 在不显著损害模型精度的前提下，比 FedCompass 更快达到目标精度，并减少 late upload、deadline trigger 和分组失配。

任何只对某一种方法改变数据分区、学习率、Q 范围、BN 行为或预算的结果，都不能进入正式对比表。

## 2. 三类数据划分必须分开命名

### 2.1 P0：FedCompass 论文式强异构设置

论文式强异构压力测试使用项目中与 FedCompass Dual Dirichlet 对齐的设置：

```yaml
partition_strategy: dirichlet_noniid
alpha1: 8.0
alpha2: 0.5
dirichlet_mode: legacy_two_level  # 未显式填写时的当前默认值
seed: 42
```

对应配置文件：

```text
examples/config/client_cifar10.yaml
```

当前实现的两层含义为：

1. 第一层决定客户端数据总量比例；8 个客户端时浓度向量约为 `[1.0] × 8`。
2. 第二层决定每个客户端的类别组成；CIFAR-10 类别均衡时浓度向量约为 `[0.05] × 10`。

因此，虽然配置写作 `alpha2=0.5`，它并不等价于常见的 class-wise `Dirichlet(0.5)`。该设置同时包含数据量异构和很强的标签异构。

P0 的用途：

- 与 FedCompass 论文 Dual Dirichlet 结果建立可比性；
- 作为强统计异构压力测试；
- 主看 TTA@50、max/last-10 accuracy、late、deadline 和 staleness。

### 2.2 P1：中强 Dual Dirichlet 后续主设置

如果 P0 下 baseline 尚未达到可用精度，可使用以下设置定位统计异构强度，但不能将它冒充为论文原始数据设置：

```yaml
alpha1: 8.0
alpha2: 2.0
dirichlet_mode: legacy_two_level
```

此时仍保留：

- 客户端数据量异构；
- 客户端标签分布异构；
- 动态计算、通信、突发延迟和 availability 异构。

P1 可以作为系统方法的开发主设置或统计异构敏感性点，但论文必须标注为 `Dual Dirichlet (8, 2)`，不能写成与 FedCompass 原论文完全一致。

对应固定配置：

```text
examples/config/client_cifar10_dual_moderate.yaml
```

自 2026-07-14 起，后续 CIFAR-10 方法主实验默认使用 P1。采用它的理由是：

- 保留 Dual Dirichlet 的客户端数据量异构；
- 保留明显的客户端标签分布异构；
- 不让极端统计异构完全掩盖本文关注的动态系统异构；
- 目标是让 FedCompass 达到论文所报告的约 55–60% 精度平台，再公平比较 StateCompass 的 TTA 和系统效率。

P1 在正式冻结前必须先通过 FedCompass seed 2026 门禁：

```text
max accuracy ≥55%
TTA@50 可达
last-10 ≥50%
max-last10 gap ≤3pp
```

若 P1 未通过，不得直接开始三方法多种子实验；应先分析曲线，再决定是否仅将 `alpha2` 调整为 3.0。调整后必须创建新实验编号，不能覆盖 P1 结果。

### 2.3 P2：标准 class-wise Dirichlet 校准设置

标准设置为：

```yaml
alpha2: 0.5
dirichlet_mode: standard
```

对应配置：

```text
examples/config/client_cifar10_standard_dirichlet.yaml
```

P2 只用于：

- 验证模型、梯度差分、评估与聚合链路能够正常收敛；
- 与常见 FL 文献中的 class-wise Dirichlet(0.5) 对齐；
- 作为较温和统计异构的敏感性实验。

P2 不是 FedCompass 论文 Dual Dirichlet 的替代品。P2 下得到的 76% 精度不能用于声称复现了论文 Dual Dirichlet 精度。

## 3. 已有数据划分诊断结果

以下均为 seed 2026、Q=40–200、SGD lr=0.1、batch size=128；600-budget 对比使用相同虚拟调度轨迹：

| 数据划分 | BN buffer | max acc | last-10 | 最后 20 点范围 | 结论 |
|---|---|---:|---:|---:|---|
| P0 Dual (8, 0.5) | legacy | 41.51% | 34.54% | 16.83pp | 精度低且震荡 |
| P0 Dual (8, 0.5) | weighted | 40.19% | 34.28% | 7.81pp | BN 改善稳定性，但未抬高平台 |
| P2 Standard (0.5) | weighted | 76.37% | 75.38% | 2.29pp | 训练链路正常，但数据更温和 |

这组结果说明：

1. weighted BN 能减少震荡，但不是 P0 低精度的充分解决方案；
2. P2 的大幅提升主要来自数据划分变得温和，不能全部归因于 BN；
3. P0 下的主要矛盾是强标签异构、大 Q、本地模型漂移、客户端参与频率失衡和等权聚合共同作用。

## 4. FedCompass 论文精度对齐要求

FedCompass 论文在 CIFAR-10 Dual Dirichlet 下报告的 top accuracy 随客户端数和设备异构设置变化。现有项目记录的参照为：

| 客户端数 | FedCompass top accuracy 参照 |
|---:|---:|
| 5 | 约 59–61% |
| 10 | 约 55–57% |

本项目当前使用 8 个客户端，介于论文的 5 与 10 之间，但不是严格相同的客户端数。因此应使用以下工程验收线，而不是声称逐点复现：

```text
FedCompass max accuracy ≥55%
TTA@50 可达
last-10 与 max 差距 ≤3pp（理想 ≤2pp）
无 NaN/Inf
```

如果 P0 下未达到该门槛，不得仅通过增加 budget 掩盖实现或训练口径问题。应依次检查：

1. 论文 batch size、模型结构、优化器和数据预处理是否一致；
2. BN buffer 与上游 FedCompass 实现的处理方式；
3. 聚合权重是否应考虑客户端样本量；
4. Q=40–200 是否在当前 8-client 强异构下造成过大的 client drift；
5. 客户端参与频率与数据类别之间是否产生系统性偏置。

注意：如果修改 Q、学习率、batch size、归一化层或聚合权重，则三种方法必须共用修改后的训练底座，并将该实验标记为“校准后实现”，不能与未修改版本混表。

## 5. 正式实验冻结项

同一组比较中，下列字段必须完全相同：

| 类别 | 冻结字段 |
|---|---|
| 数据 | dataset、partition strategy、alpha1、alpha2、partition seed、客户端数 |
| 模型 | ResNet-18 结构、归一化层、初始化 seed |
| 本地训练 | optimizer、lr、momentum、weight decay、batch size、梯度裁剪 |
| 联邦训练 | Qmin、Qmax、budget、staleness function、server alpha、聚合权重 |
| 系统环境 | 8-client profiles、runtime seed、模型/更新大小、单步时间校准 |
| 评估 | 同一 CIFAR-10 test set、相同评估频率、相同 TTA 定义 |

只允许变化的方法字段为：

```text
FedCompass：原始速度驱动 Q 与分组
Trust-Q：状态感知 Q，原分组不变
Q + Group：状态感知 Q + 状态时间窗分组
```

## 6. “新方法优于 FedCompass”的判定

StateCompass 不需要以更高 final accuracy 作为唯一胜出条件。正式门禁为：

### 6.1 精度安全门

```text
StateCompass last-10 ≥ FedCompass last-10 −1pp
StateCompass max accuracy ≥ FedCompass max accuracy −1pp
```

若低于 1pp 以上，暂不扩展多 seed。

### 6.2 系统与收敛收益门

至少满足以下主要条件：

- TTA@50 优于 FedCompass；
- accuracy-time AUC 不低于 FedCompass；
- late uploads 明显减少；
- deadline triggers 明显减少；
- mean/max staleness 不发生不可接受的恶化；
- Q+Group 不产生严重组碎片化。

论文应表述为：

> 在达到相当模型质量的条件下，StateCompass 通过状态感知工作量分配与时间窗分组，改善动态端侧异构环境中的时间到精度和调度可靠性。

不应为了得到“新方法优于 baseline”的结论而：

- 给新方法使用更温和的数据划分；
- 给新方法单独降低学习率或 Qmax；
- 只报告有利的单个 seed；
- 只比较 final accuracy 而忽略 max、last-10、TTA 和 AUC；
- 将 P2 标准 Dirichlet 结果描述为 P0 论文 Dual Dirichlet 结果。

## 7. 推荐实验顺序

### Stage A：P1 baseline 精度门禁

目标：先让 FedCompass 在中强 Dual Dirichlet 下达到论文参照的可用精度。

```text
P1 Dual (8, 2)
FedCompass
seed 2026
短预算 600 用于筛选
正式预算在曲线分析后冻结
```

P1 通过上述精度门禁后，冻结数据与训练底座，再进入三方法比较。

### Stage B：三方法单 seed 门禁

使用完全相同的 P1 数据、模型、优化器、聚合器、Q 范围和预算运行 FedCompass、Trust-Q V2 与 Q+Group。

### Stage C：多 seed 正式实验

单 seed 门禁通过后，在冻结的 P1 训练底座上运行：

```text
FedCompass、Trust-Q V2、Q + Group
seeds = 2026, 2027, 2028
```

报告均值、标准差和逐 seed 胜负，不允许只选择最佳 seed。

### Stage D：论文式强异构压力测试

主实验通过后，在 P0 Dual `(8, 0.5)` 下至少运行 seed 2026，报告三种方法在极端统计异构下的退化，不要求其达到与 P1 相同的精度平台。

### Stage E：统计异构敏感性

建议至少包含：

```text
P2 Standard Dirichlet(0.5)：温和/常见口径
P1 Dual (8, 2)：中强异构
P0 Dual (8, 0.5)：论文式强异构
```

## 8. 每次运行前后的检查清单

运行前：

- [ ] 固定或归档代码版本；
- [ ] 输出目录为空或明确使用 `--force`；
- [ ] 三种方法的 client/server config 相同；
- [ ] 记录 alpha1、alpha2、dirichlet_mode 和 partition seed；
- [ ] 记录 Qmin/Qmax、lr、batch size 和 BN buffer 模式；
- [ ] 保持 RTX 4090 校准参数不变。

运行后：

- [ ] `experiment_config.json` 中 resolved training base 与预期一致；
- [ ] budget 计数与 aggregation trace 一致或解释差异；
- [ ] 计算 final、max、last-10、TTA@30/40/45/50 和 AUC；
- [ ] 计算 late、deadline、mean/max staleness 和累计 local steps；
- [ ] 检查客户端更新数和训练量是否严重失衡；
- [ ] 检查后期震荡和 max-last10 差距；
- [ ] 保留完整日志、trace、配置和代码版本标识。

## 9. 当前决策记录

截至 2026-07-14：

1. P2 Standard Dirichlet(0.5) + weighted BN 已达到 max 76.80%，证明训练链路可收敛；
2. P0 Dual (8, 0.5) + weighted BN 在 budget 600 下 max 40.19%，未达到论文参照；
3. 暂不把 P2 结果当作 FedCompass 论文复现结果；
4. 后续主实验使用 P1 Dual `(8, 2)`；P0 Dual `(8, 0.5)` 保留为论文式强异构压力测试；
5. 无论选择 P0、P1 或 P2，同一张方法比较表中的三种方法必须使用完全相同的数据与训练底座。
